# Week 7 학습 회고 — 멀티 LLM 파이프라인

## 5일간의 여정 요약

이번 주는 MCP를 활용한 멀티 LLM 파이프라인을 구축하며 AI 서비스 개발의 전체 흐름을 직접 체험했습니다.
텍스트 → 장면 추출 → 이미지 생성 → 영상 생성으로 이어지는 파이프라인을 구현하면서,
LLM을 단순히 사용하는 것을 넘어 여러 모델을 조율하는 오케스트레이터 역할까지 다뤘습니다.

## Day별 학습 회고

### Day 1 — LLM API 기초와 프롬프트 설계
- OpenAI API 연결 및 시스템 프롬프트 설계 원리 학습
- **배운 점**: 시스템 프롬프트가 결과 품질을 결정짓는 핵심임을 체감
- **어려웠던 점**: JSON 구조화 출력을 안정적으로 받는 방법

### Day 2 — 이미지 생성 모델 (DALL-E 3 & Flux)
- DALL-E 3와 fal.ai Flux 모델 비교 실습
- **배운 점**: 동일한 프롬프트도 모델별로 스타일이 크게 다름
- **어려웠던 점**: 프롬프트 어휘(shot, angle, lighting)가 결과에 미치는 영향 파악

### Day 3 — 멀티모달 파이프라인 설계
- 텍스트 → 이미지 파이프라인 연결 구조 설계
- **배운 점**: `agents/` 폴더로 모듈화하면 각 단계를 독립적으로 교체 가능
- **어려웠던 점**: 모듈 간 인터페이스(반환 타입) 통일

### Day 4 — 도메인별 프롬프트 특화
- 여행·제품·이모티콘 도메인별 프롬프트 어휘 실습
- **배운 점**: 동일 파이프라인도 도메인 프롬프트에 따라 완전히 다른 결과
- **어려웠던 점**: golden_hour / blue_hour 같은 시각 어휘를 찾고 검증하는 과정

### Day 5 — 최종 파이프라인 완성 및 제출
- `picture_diary_pipeline()` 완성, 도메인 응용, 채점 규격 준수
- **배운 점**: 파일명·함수명 규칙이 실제 협업과 자동화 채점에서 얼마나 중요한지
- **어려웠던 점**: 비동기 영상 생성 폴링 구현

## 가장 인상 깊었던 것

`golden_hour` vs `blue_hour` 비교 실험이 가장 인상적이었습니다.
프롬프트 한 단어 차이로 이미지 전체의 색감·분위기·감정이 완전히 바뀌는 것을 보며,
프롬프트 엔지니어링이 단순한 텍스트 작성이 아닌 **시각적 언어**임을 깨달았습니다.
